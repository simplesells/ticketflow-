## Context

用户通过表单创建工单。后端接收 JSON 数据，生成唯一工单号，以"待处理"状态入库。

## Goals / Non-Goals

**Goals:**
- 用户可填写标题、描述、类型、优先级并提交
- 前后端各做必填校验
- 成功后跳转到详情页

**Non-Goals:**
- 文件上传
- 草稿保存
- 列表展示（后续变更）

## Decisions

| # | 决定 | 理由 |
|---|------|------|
| 1 | 工单号格式 WO-YYYYMMDD-NNNN | 可读、唯一、基于日期+计数 |
| 2 | 新工单默认"待处理" | 符合业务流转起点 |
| 3 | POST 成功返回 201 + 完整工单对象 | 前端可直接跳转使用 |

## Risks / Trade-offs

- [Risk] 工单号在高并发下可能重复 → 当前 Demo 单用户，不考虑
