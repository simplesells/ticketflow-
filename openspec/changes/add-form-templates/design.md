## Context

当前四个必填字段（标题、描述、类型、优先级）全部手填。为常见工单类型预设模板，减少输入量。

## Goals / Non-Goals

**Goals:**
- 预设常用模板，选模板后自动填充表单
- 调度者可以管理模板（增删改）
- 模板只影响表单默认值，不影响提交后的工单

**Non-Goals:**
- 模板不和状态机绑定（这是方向 A 的事）
- 不强制使用模板，用户可以忽略手动填

## Decisions

1. **模板只存默认值，不改变工单结构**
   - 工单创建后和模板无关联
   - 避免 `workorders` 表加外键

2. **调度者管理模板**
   - 调度者是流程管理者，自然该管模板
   - API 加角色守卫 `requireRole(['dispatcher'])`

3. **种子数据 4 个模板**
   - 打印机故障（报修/高）
   - 设备申请（咨询/中）
   - 网络故障（报修/紧急）
   - 环境投诉（投诉/中）
